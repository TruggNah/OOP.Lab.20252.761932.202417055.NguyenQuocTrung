(?) If the passing object is not an instance of Media, what happens?

When overriding the equals() method, the Object parameter must be cast to the appropriate class type before comparing attributes.
Therefore, we should first check whether the passing object is an instance of the expected class by using the instanceof operator.
If the passing object is not an instance of Media, the method should return false.
Otherwise, directly casting the object may cause a ClassCastException at runtime.
This checking step helps ensure type safety and prevents invalid object comparisons.

-> The Media class should implement the Comparable<Media> interface because all media objects in the cart are stored and managed through the Media type.
-> The compareTo() method should compare media objects according to the desired ordering rule. For example, it can first compare titles, and if the titles are equal, compare costs.
-> No. Using Comparable only allows one default ordering rule for a class because a class can only have one compareTo() implementation. Therefore, it is not suitable when we need multiple sorting rules such as “title then cost” and “cost then title”.
-> If DVDs require a different ordering rule, we should use Comparator instead of only Comparable. We can create a separate comparator for DVDs that compares by title, then decreasing length, then cost, while other media types continue using different comparators. This approach is more flexible and extensible.
